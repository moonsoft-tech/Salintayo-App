B. Data Design
Figure 9
Entity Relationship Diagram (ERD)
Figure 9 illustrates the structured design of the 
SalinTayo
 language-learning application composed of interconnected entities that support user interaction, language content management, and chat monitoring. The core of the system revolves around the users table, 
which stores user information and connects to multiple modules such as 
chatSessions
, notifications, and dialect preferences. Each user can have multiple chat sessions, receive notifications, and maintain learning progress and streak history, establishing one-to-many relationships across these entities.
The dialects table serves as another central component, containing detailed information about Philippine languages and regional variations. Each dialect can be associated with multiple users who select it as their preferred learning language. The dialects table includes language codes, native names, speaker counts, regions, and cultural metadata to support comprehensive language learning.
The 
chatSessions
 table captures user conversations with the AI assistant, including message history, timestamps, and the dialect being used. Each chat session belongs to a single user and records interaction history for user reference and potential AI training. The notifications table ensures communication by delivering updates and alerts related to learning reminders, streak alerts, level-up messages, and community notifications.
On the administrative side, additional tables such as 
admin_user
, 
activityLog
, and 
ai_training
 are used to manage system operations. The 
admin_user
 table stores administrator account information linked to users through a foreign key relationship. The 
activityLog
 table maintains an audit trail of admin actions including user management, dialect updates, and settings changes. The 
ai_training
 table stores phrases from user interactions where the AI model performed 
suboptimally
, along with corrections and training status, supporting continuous improvement of the language model. These entities together ensure efficient system management and comprehensive tracking of all system operations.
